# writer-post — X投稿 作成スキル

## いつ使うか
ユーザーが「X投稿を作って」「ポスト作って」「この記事の告知ポスト書いて」と言ったとき。

## 前提
- 開始前に CLAUDE.md の「人格・ブランド定義」「承認ルール」を読む。
- 入力は2系統: ①atoms.csv の単体ネタ ②drafts/ の完成note記事（告知ポスト）。

## 投稿フォーマット
- 全角140字以内（URL分は別カウントでよい）
- 構成: フック1行 → 要点 → 行動喚起 or 問いかけ1行
- ハッシュタグは最大2個（例: #生成AI #ClaudeCode）
- 1ネタにつき2パターン作る（A: 事実訴求 / B: 問いかけ訴求）

## トーンルール
- CLAUDE.md準拠。煽り・誇張・絵文字の多用をしない。
- note記事の告知の場合は末尾に記事URLのプレースホルダ `（記事URL）` を置く。

## 手順
1. 対象（atom or 記事）を特定する
2. 上記フォーマットでA/B 2案を作る
3. `drafts/YYYY-MM-DD-x-posts.md` に追記出力する（既存があれば上書きせず追記）
4. `pipeline.csv` に行を追加（channel=x, status=drafted, planned_date=今日）
5. ユーザーに2案を提示し、「実投稿は手動でお願いします」と添える（承認ルール遵守）

## 品質チェック（Output Gate）
- [ ] 140字以内
- [ ] 事実に出典がある（誤情報を断定していない）
- [ ] 誇張語・過剰な絵文字を使っていない
- [ ] 実投稿は自動で行っていない（下書きのみ）

## 出力先
- 投稿案: `drafts/`（Markdown）
- 状態更新: `data/pipeline.csv`
