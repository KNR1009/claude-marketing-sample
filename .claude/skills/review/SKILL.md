# review — 事実確認・品質レビュースキル

## いつ使うか
記事やX投稿の下書きができたあと、「レビューして」「ファクトチェックして」「公開して大丈夫か見て」と言われたとき。

## 前提
- 開始前に CLAUDE.md の「品質基準(Output Gate)」を読む。
- 対象は `drafts/` の最新ファイル、または指定ファイル。
- 関連atomの出典URL（data/atoms.csv）を突き合わせる。

## レビュー観点（CLAUDE.md品質基準と一致させる）
1. 事実: 数字・固有名詞・日付・機能名が一次情報と一致するか
2. 出典: 各主張に出典URLがあるか。URLが生きているか（必要ならWeb確認）
3. 帰属: 「誰の発表か」を取り違えていないか（Claude/Gemini/OpenAIの混同に特に注意）
4. 誇張: 元ソースが言っていないことを断定で書いていないか
5. 重複: outputs.csv の過去公開と焼き直しになっていないか

## 手順
1. 対象ファイルとひも付くatomを読む
2. 上記5観点で1項目ずつ判定し、問題箇所を「指摘 → 該当箇所の引用 → 修正案」の形で列挙する
3. 総合判定を出す: PASS（公開可） / FIX（要修正） / BLOCK（出典不足で公開不可）
4. PASSの場合のみ、対象の pipeline.csv 行の status を `reviewed` に更新する
5. 公開後にユーザーが outputs.csv へ記録できるよう、記入用の1行を提示する

## 出力
- レビュー結果（チャットに表示） + 修正が必要なら修正案
- PASS時のみ `data/pipeline.csv` を更新

## 注意（承認ルール）
- このスキルは公開操作をしない。判定と記録更新のみ。実投稿・公開はユーザーが行う。
